OVERVIEW

AI 에이전트란 무엇인가?

스스로 판단하고 행동하는 AI

AI 에이전트는 단순한 챗봇을 넘어, 목표를 설정하고 계획을 세우고 도구를 사용해 실행하는 자율적 AI 시스템입니다.

기존 AI가 "질문-답변" 1회성 구조라면, 에이전트는 관찰 - 판단 - 행동 - 반성의 루프를 반복합니다.

  • 파일 시스템, 웹, API 등 외부 환경을 직접 탐색
  • 수집한 정보를 바탕으로 다음 행동을 자율 결정
  • 코드 실행, 파일 편집 등 실제 작업을 수행
  • 결과를 확인하고 부족하면 스스로 수정
ARCHITECTURE

에이전트 구조 vs 전통 AI

전통 AI 방식

사용자 질문 -> AI 답변

매번 사람이 지시해야 하며, 컨텍스트를 유지하지 못합니다. 도구 사용이 제한적이고 단일 턴으로 끝납니다.

AI 에이전트 방식

목표 설정 -> 계획 -> 도구 사용 -> 실행 -> 검증

자율적으로 여러 단계를 거쳐 작업을 완수합니다. 실패하면 스스로 수정하고 다시 시도합니다.

  • MCP를 통한 도구 연결
  • 메모리와 컨텍스트 유지
  • 반복적 자기 개선 루프
CORE CONCEPTS

AI 에이전트의 4대 핵심 원리

01
환경 인식 (Perception)

파일, 웹, API 등 외부 환경을 탐색하고 정보를 수집하는 능력

02
자율 판단 (Reasoning)

수집한 정보를 분석하여 최적의 행동 방침을 스스로 결정

03
도구 활용 (Tool Use)

코드 실행, 파일 편집, 외부 서비스 호출 등 실제 작업을 수행

04
반복 학습 (Reflection)

결과를 평가하고 부족한 부분을 수정하며 목표 달성까지 반복
